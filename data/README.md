# JCC/JCG データ構造

## location-data.json の構造

```json
{
  "version": "1.0.0",
  "lastUpdate": "2024-02-03",
  "locations": [
    {
      "lat": 35.6895,
      "lon": 139.6917,
      "prefecture": "東京都",
      "city": "千代田区",
      "jcc": "1001",
      "jcg": "10001"
    }
  ]
}
```

## フィールドの説明

- `version`: データフォーマットのバージョン
- `lastUpdate`: 最終更新日（YYYY-MM-DD形式）
- `locations`: ロケーションデータの配列

### ロケーションオブジェクト

- `lat`: 緯度（10進数形式）
- `lon`: 経度（10進数形式）
- `prefecture`: 都道府県名
- `city`: 市区町村名
- `jcc`: JCC（Japan Century City）コード
- `jcg`: JCG（Japan Century Gun）コード

## データの追加方法

### 手動追加

`locations` 配列に新しいオブジェクトを追加してください。

```json
{
  "lat": 緯度,
  "lon": 経度,
  "prefecture": "都道府県名",
  "city": "市区町村名",
  "jcc": "JCCコード",
  "jcg": "JCGコード"
}
```

### データソース

JCC/JCGコードは以下から取得できます：

- JARL（一般社団法人日本アマチュア無線連盟）公式サイト
- JCC/JCGリスト（PDF等で公開されている場合）
- アマチュア無線関連のデータベース

### 推奨データポイント

精度を上げるため、各市区町村の代表地点を追加することを推奨します：

1. 市区町村役場の位置
2. 主要な山岳地点（SOTA運用に便利）
3. 境界付近の地点（判定精度向上のため）

## データの検証

以下の点に注意してデータを検証してください：

1. 緯度は -90 〜 90 の範囲
2. 経度は -180 〜 180 の範囲
3. JCC/JCGコードは正しい形式（数字4-5桁）
4. 都道府県名・市区町村名は正式名称

## より高度なデータ構造（将来の拡張）

将来的には以下のような拡張も検討できます：

```json
{
  "lat": 35.6895,
  "lon": 139.6917,
  "prefecture": "東京都",
  "city": "千代田区",
  "jcc": "1001",
  "jcg": "10001",
  "elevation": 25,           // 標高（メートル）
  "gl": "PM95",              // グリッドロケーター
  "bounds": {                // 境界情報
    "north": 35.7,
    "south": 35.68,
    "east": 139.7,
    "west": 139.68
  }
}
```
